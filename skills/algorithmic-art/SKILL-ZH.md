算法哲学是一种计算美学运动，然后通过代码来表达。输出.md文件（哲学理念）、.html文件（交互式查看器）和.js文件（生成算法）。

这个过程分两步进行：
1. 算法哲学创建（.md文件）
2. 通过创建p5.js生成艺术来表达（.html + .js文件）

首先，执行以下任务：

## 算法哲学创建

首先，创建一个**视觉哲学**（不是静态图像或模板），将通过以下方式被诠释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪声场、有机系统
- 粒子、流场、力场
- 参数化变化和控制混沌

### 关键理解
- **接收什么**：用户提供的微妙输入或说明，作为基础但不应限制创意自由
- **创建什么**：算法哲学/生成美学运动
- **接下来会发生什么**：同一版本接收哲学理念并用**代码表达**——创建90%算法生成、10%必要参数的p5.js草图

考虑这种方法：
- 为生成艺术运动撰写宣言
- 下一阶段涉及编写使其具象化的算法

哲学必须强调：算法表达。涌现行为。计算之美。种子变化。

### 如何生成算法哲学

**命名这个运动**（1-2个词）："有机湍流" / "量子谐波" / "涌现静默"

**阐述哲学**（4-6段——简洁但完整）：

为了捕捉**算法本质**，表达这个哲学如何通过以下方式显现：
- 计算过程和数学关系？
- 噪声函数和随机模式？
- 粒子行为和场动力学？
- 时间演化和系统状态？
- 参数化变化和涌现复杂性？

**关键指南**：
- **避免冗余**：每个算法方面只提及一次。除非添加新的深度，否则避免重复关于噪声理论、粒子动力学或数学原理的概念。
- **反复强调工艺水平**：哲学必须**多次强调**最终算法应该看起来花费了无数小时开发，经过精心优化，来自计算美学领域绝对顶尖的专家。这种框架至关重要——重复使用如"精心制作的算法"、"深度计算专业知识的产物"、" painstaking优化"、"大师级实现"等短语。
- **保留创意空间**：对算法方向保持足够具体，但足够简洁，以便下一个Claude能够在极高的工艺水平上进行诠释性的实现选择。

哲学必须指导下一个版本**算法地**表达想法，而不是通过静态图像。美丽存在于过程中，而不是最终帧中。

### 哲学示例

**"有机湍流"**
哲学：混沌受自然法则约束，秩序从无序中涌现。
算法表达：流动场由分层的Perlin噪声驱动。数千个粒子沿着向量力移动，它们的轨迹累积成有机密度图。多个噪声八度创建湍流区域和平静区域。颜色从速度和密度中涌现——快速粒子燃烧明亮，缓慢粒子褪入阴影。算法运行直到达到平衡——这是经过无数次迭代精心调整的平衡，每个参数都由计算美学大师完善。

**"量子谐波"**
哲学：离散实体表现出波状干涉模式。
算法表达：粒子在网格上初始化，每个携带通过正弦波演化的相位值。当粒子靠近时，它们的相位发生干涉——相长干涉创建明亮节点，相消创建空隙。简谐运动生成复杂的涌现曼陀罗。这是经过辛苦的频率校准的结果，每个比率都被精心选择以产生共振之美。

**"递归低语"**
哲学：跨尺度的自相似性，有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支稍微随机化但受黄金比例约束。L系统或递归细分生成树状形式，感觉既数学又有机。微妙的噪声扰动打破完美对称。线条权重随每个递归级别递减。每个分支角度都是深度数学探索的产物。

**"场动力学"**
哲学：不可见的力量通过它们对物质的影响变得可见。
算法表达：由数学函数或噪声构建的向量场。粒子在边缘诞生，沿着场线流动，当达到平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化只显示轨迹——不可见力量的幽灵般证据。计算舞蹈通过力量平衡精心编排。

**"随机结晶"**
哲学：随机过程结晶成有序结构。
算法表达：随机圆填充或Voronoi镶嵌。从随机点开始，让它们通过松弛算法演化。细胞相互推挤直到达到平衡。颜色基于细胞大小、邻居数量或到中心的距离。有机镶嵌涌现的感觉既随机又必然。每个种子产生独特的晶体之美——大师级生成算法的标志。

*这些是压缩的示例。实际的算法哲学应该是4-6个实质性的段落。*

### 基本原则
- **算法哲学**：创建一个通过代码表达的计算世界观
- **过程胜于产品**：始终强调美丽从算法的执行中涌现——每次运行都是独特的
- **参数化表达**：想法通过数学关系、力、行为交流——不是静态构图
- **艺术自由**：下一个Claude算法地解释哲学——提供创意的实现空间
- **纯粹生成艺术**：这是关于制作**活算法**，而不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，经过无数次优化，是计算美学领域绝对顶尖专家的深度专业知识的产物

**算法哲学应该是4-6段长。**用诗意的计算哲学填充它，将预期愿景汇聚在一起。避免重复相同的观点。将这个算法哲学输出为.md文件。

**关键步骤**：在实现算法之前，从原始请求中识别微妙的概念线索。

**基本原则**：

### 技术要求

**种子随机性（艺术块模式）**：
```javascript
// 始终使用种子以实现可重现性
let seed = 12345; // 或从用户输入哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构——遵循哲学**：

为了建立从算法哲学自然涌现的参数，考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包括种子以实现可重现性
  // 颜色
  // 添加控制算法的参数：
  // - 数量（多少？）
  // - 比例（多大？多快？）
  // - 概率（多可能？）
  // - 比率（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（什么时候行为改变？）
};
```

**设计有效参数，专注于系统需要可调谐的属性，而不是考虑"模式类型"。**

**核心算法——表达哲学**：

**关键**：算法哲学应该决定构建什么。

为了通过代码表达哲学，避免思考"我应该使用哪个模式？"而是思考"如何通过代码表达这个哲学？"

如果哲学是关于**有机涌现**，考虑使用：
- 随时间累积或增长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果哲学是关于**数学之美**，考虑使用：
- 几何关系和比率
- 三角函数和谐波
- 产生意外模式的精确计算

如果哲学是关于**控制混沌**，考虑使用：
- 严格边界内的随机变化
- 分叉和相变
- 从无序中涌现的有序

**算法从哲学流动，而不是从选项菜单。**

为了指导实现，让概念本质告知创造性的原创选择。为这个特定请求构建表达愿景的东西。

**画布设置**：标准p5.js结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化你的系统
}

function draw() {
  // 你的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：为了达到大师级，创建感觉经历了无数次迭代由大师级生成艺术家的算法。仔细调整每个参数。确保每个模式都是有目的地涌现。这不是随机噪声——这是通过深度专业知识精炼的**控制混沌**。

- **平衡**：复杂性而没有视觉噪音，有序而没有僵化
- **颜色和谐**：深思熟虑的调色板，而不是随机RGB值
- **构图**：即使在随机性中，保持视觉层次和流动
- **性能**：平滑执行，如果动画则优化实时
- **可重现性**：相同种子**总是**产生相同输出

### 输出格式

输出：
1. **算法哲学**——作为markdown或文本解释生成美学
2. **单个HTML工件**——从 `templates/viewer.html` 构建的自我包含的交互式生成艺术（参见第0步和下一节）

HTML工件包含一切：p5.js（来自CDN）、算法、参数控件和UI——在一个文件中，可以在 claude.ai 工件或任何浏览器中立即工作。从模板文件开始，而不是从零开始。

## 交互式工件创建

**提醒**：`templates/viewer.html` 应该已经被阅读（参见第0步）。使用该文件作为起点。

为了允许探索生成艺术，创建一个单一的、自我包含的HTML工件。确保这个工件在 claude.ai 或任何浏览器中立即工作——无需设置。所有内容都内联嵌入。

### 关键：什么是固定vs可变

`templates/viewer.html` 文件是基础。它包含确切的结构和样式需要。

**固定（始终完全按所示包括）**：
- 布局结构（头部、侧边栏、主画布区域）
- Anthropic品牌（UI颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + 转到按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每个艺术品自定义）**：
- 整个p5.js算法（setup/draw/类）
- 参数对象（定义艺术品需要什么）
- 侧边栏中的参数部分：
  - 参数控件数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控件类型（滑块、输入等）
- 颜色部分（可选）：
  - 一些艺术品需要颜色选择器
  - 一些艺术品可能使用固定颜色
  - 一些艺术品可能是单色的（不需要颜色控件）
  - 根据艺术品的需要决定

**每个艺术品应该有独特的参数和算法！** 固定部分提供一致的UX——其他一切表达独特的愿景。

### 必需功能

**1. 参数控件**
- 数字参数的滑块（粒子数、噪声比例、速度等）
- 调色板的颜色选择器
- 参数更改时实时更新
- 重置按钮恢复默认值

**2. 种子导航**
- 显示当前种子号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮随机种子
- 输入字段跳转到特定种子
- 请求时生成100个变体（种子1-100）

**3. 单个工件结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js来自CDN——始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联——干净、最小 */
    /* 画布在顶部，控件在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控件 -->
  </div>
  <script>
    // 所有p5.js代码在这里内联
    // 参数对象、类、函数
    // setup()和draw()
    // UI处理器
    // 一切自我包含
  </script>
</body>
</html>
```

**关键**：这是一个单一工件。没有外部文件，没有导入（除了p5.js CDN）。一切内联。

**4. 实现细节——构建侧边栏**

侧边栏结构：

**1. 种子（固定）**——始终完全按所示包括：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）——为艺术品创建控件**：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
添加尽可能多的control-group div作为参数数量。

**3. 颜色（可选/可变）**——如果艺术品需要可调整颜色则包括：
- 如果用户应该控制调色板则添加颜色选择器
- 如果艺术品使用固定颜色则跳过此部分
- 如果艺术品是单色的则跳过
- 根据艺术品的需要决定

**4. 操作（固定）——始终完全按所示包括**：
- 重新生成按钮
- 重置按钮
- 下载PNG按钮

**要求**：
- 种子控件必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数必须有UI控件
- 重新生成、重置、下载按钮必须工作
- 保持Anthropic品牌（UI样式，不是艺术品颜色）

### 使用工件

HTML工件立即工作：
1. **在 claude.ai 中**：显示为交互式工件——立即运行
2. **作为文件**：保存并在任何浏览器中打开——不需要服务器
3. **分享**：发送HTML文件——它完全自我包含

## 变体与探索

工件默认包括种子导航（上一步/下一步/随机按钮），允许用户探索变体而无需创建多个文件。如果用户想要突出显示特定变体：

- 包括种子预设（"变体1：种子42"、"变体2：种子127"等按钮）
- 添加"画廊模式"并排显示多个种子的缩略图

这就像从同一块印版创建一系列印刷品——算法是一致的，但每个种子揭示了其潜力的不同方面。交互性质意味着用户通过探索种子空间发现自己的最爱。

## 创作过程

**用户请求** → **算法哲学** → **实现**

每个请求都是独特的。这个过程涉及：

1. **解释用户的意图**——正在寻求什么美学？
2. **创建一个算法哲学**（4-6段）描述计算方法
3. **用代码实现**——构建表达这个哲学的算法
4. **设计适当的参数**——什么是可调的？
5. **构建匹配的UI控件**——这些参数的滑块/输入

**常量**：
- Anthropic品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自我包含HTML工件

**其他一切都是可变的**：
- 算法本身
- 参数
- UI控件
- 视觉结果

为了获得最佳结果，相信创造力，让哲学指导实现。
